# Rookie Tools - 项目说明文档

## 一、项目简介

**Rookie Tools** 是一款专为 AI 绘画爱好者打造的本地离线 Prompt 工具箱，支持 Stable Diffusion WebUI、ComfyUI 和 NovelAI 等主流 AI 绘画工具的用户使用。

项目名称中的 "Rookie" 寓意着帮助新手用户快速上手 AI 绘画，同时也是开发者对自己技术成长的一种期许。

---

## 二、核心功能

### 2.1 魔法解析 🪄

将 AI 生成的图片拖入或点击上传，自动解析图片中嵌入的生成参数。

**支持的生成器：**
- **ComfyUI**：解析 workflow 和 prompt 元数据
- **Stable Diffusion WebUI (AUTOMATIC1111)**：解析 parameters 字段
- **NovelAI**：解析 Comment 字段

**解析内容：**
- 正向提示词（Positive Prompt）
- 负向提示词（Negative Prompt）
- 生成参数（采样器、步数、CFG、种子、尺寸等）
- 模型信息（模型名称、哈希值）
- 高清修复参数（放大倍数、重绘幅度等）

**使用流程：**
```
上传图片 → 自动识别生成器类型 → 提取参数 → 展示结果
                                           ↓
                       ┌───────────────────┼───────────────────┐
                       ↓                   ↓                   ↓
                    复制提示词         导入提示词管理        查看历史记录
```

---

### 2.2 标签选择器 🏷️

通过分类标签快速构建提示词，支持权重调整。

**七类标签分类：**

| 分类 | 说明 | 示例 |
|-----|------|------|
| 画师 | 艺术家风格 | Makoto Shinkai, Greg Rutkowski |
| 风格 | 艺术风格 | anime style, watercolor, 3d render |
| 质量 | 画面质量 | masterpiece, best quality, highly detailed |
| 构图 | 画面构图 | close-up, full body, from above |
| 光影 | 光线效果 | cinematic lighting, soft light |
| 角色 | 角色特征 | 1girl, smile, long hair |
| 环境 | 场景环境 | outdoors, night sky, cityscape |

**功能特性：**
- 🔍 **智能搜索**：支持标签名称和别名匹配
- ⚖️ **权重调整**：点击 +/- 调整标签权重（0.1 - 2.0）
- 📋 **一键复制**：将选中标签格式化为提示词格式
- 📊 **使用统计**：根据使用频率智能排序

**标签格式示例：**
```
无权重: masterpiece, best quality
有权重: (masterpiece:1.2), (best quality:1.4)
```

---

### 2.3 提示词管理 📝

集中管理你的提示词库，支持创建、编辑、删除和快速复制。

**主要功能：**
- ✏️ **创建提示词**：手动编写正向/负向提示词
- 🔄 **编辑修改**：随时调整已有提示词内容
- 🗑️ **删除管理**：清理不再需要的提示词
- 🔍 **搜索查找**：按名称或内容快速定位
- 📋 **批量复制**：支持复制正向、负向或全部内容

**数据来源：**
- 手动创建
- 从魔法解析模块导入

**提示词结构：**
```
提示词记录
├── 名称（如：赛博朋克风格）
├── 正向提示词（期望出现的内容）
├── 负向提示词（期望避免的内容）
├── 关联标签
├── 来源标记（手动/解析导入）
└── 创建/更新时间
```

---

### 2.4 画师管理 🎨

整理和收藏你喜欢的画师风格，支持创建画师串组合多种风格。

**功能特性：**
- 📚 **画师库**：集中管理画师资源
- ⭐ **收藏功能**：标记常用画师
- 🔢 **使用统计**：记录使用频率
- 🔗 **画师串**：组合多个画师风格
- 📥 **批量导入**：从文本文件批量导入画师列表

**画师串概念：**
画师串是一种将多个画师风格组合在一起的便捷方式。例如创建一个 "日系动漫风" 的画师串，可以包含多位擅长日系风格的画师，使用时一键插入所有画师标签。

## 三、技术架构

### 3.1 技术栈概览

| 层级 | 技术 | 用途 |
|-----|------|------|
| 框架 | Vue 3.5 | 用户界面构建 |
| 语言 | TypeScript | 类型安全开发 |
| 构建 | Vite 6 | 项目构建与开发服务器 |
| 状态 | Pinia 2 | 全局状态管理 |
| 样式 | Tailwind CSS 3.4 | 原子化样式 |
| 存储 | IndexedDB | 本地离线数据存储 |
| 图标 | Lucide Vue | 图标组件库 |

### 3.2 项目结构

```
rookie_tools/
├── src/
│   ├── modules/              # 功能模块
│   │   ├── spell-parser/     # 魔法解析模块
│   │   ├── tag-selector/     # 标签选择器模块
│   │   ├── prompt-manager/   # 提示词管理模块
│   │   └── artist-manager/   # 画师管理模块
│   ├── components/           # 通用组件
│   ├── services/             # 服务层
│   │   ├── db.ts            # IndexedDB 服务
│   │   ├── spellParser.ts   # 图片解析服务
│   │   └── tagLoader.ts     # 标签加载服务
│   ├── stores/               # Pinia 状态管理
│   │   ├── spell.ts         # 解析模块状态
│   │   ├── tag.ts           # 标签模块状态
│   │   ├── prompt.ts        # 提示词模块状态
│   │   └── artist.ts        # 画师模块状态
│   ├── composables/          # 组合式函数
│   ├── types/                # TypeScript 类型定义
│   ├── utils/                # 工具函数
│   ├── App.vue              # 根组件
│   └── main.ts              # 入口文件
├── data/lists/               # 标签数据文件
├── public/                   # 静态资源
└── index.html               # HTML 入口
```

### 3.3 数据存储方案

本项目采用 **IndexedDB** 作为本地离线存储方案，所有数据均保存在用户浏览器中，不上传至任何服务器。

**存储的数据类型：**

| 数据类型 | 说明 | 存储方式 |
|---------|------|---------|
| 标签数据 | 七类标签的完整列表 | IndexedDB |
| 提示词记录 | 用户创建的提示词 | IndexedDB |
| 画师数据 | 画师信息和收藏状态 | IndexedDB |
| 画师串 | 画师组合数据 | IndexedDB |
| 解析历史 | 解析过的图片记录 | IndexedDB（限制50条）|
| 主题偏好 | 深色/浅色模式设置 | localStorage |

**数据安全：**
- 完全离线，无需网络连接
- 数据不出本地设备
- 支持浏览器隐私模式

## 五、使用指南

1. **安装依赖**
   ```
   npm install
   ```

2. **启动开发服务器**
   ```
   npm run dev
   ```

3. **构建生产版本**
   ```
   npm run build
   ```

### 5.2 数据准备

标签数据从 `data/lists/` 目录下的文本文件加载：
- `artists.txt` - 画师标签
- `style.txt` - 风格标签
- `quality.txt` - 质量标签
- `composition.txt` - 构图标签
- `lighting.txt` - 光影标签
- `character.txt` - 角色标签
- `environment.txt` - 环境标签

**文件格式：**
```
标签名称 | 别名1,别名2,别名3
示例：
masterpiece | 杰作,最佳质量
1girl | 单人,女孩
```

## 六、特色设计

### 6.1 离线优先

所有功能无需联网即可使用，数据完全存储在本地，保护用户隐私。

### 6.2 生成器兼容

智能识别多种 AI 绘画工具的元数据格式：
- ComfyUI 的 workflow/prompt JSON 格式
- WebUI 的 parameters 文本格式
- NovelAI 的 Comment JSON 格式

### 6.3 图片压缩

解析的图片会自动生成两种尺寸：
- **缩略图**：120x120，用于历史列表展示
- **预览图**：保持原比例压缩，用于详情页展示

有效节省存储空间，限制历史记录最多保存 50 条。

### 6.4 使用频率统计

标签和画师支持使用频率统计，常用项会自动排在前面，提升查找效率。
