# Rookie Tools - 项目说明文档

## 一、项目简介

**Rookie Tools** 是一款专为 AI 绘画爱好者打造的本地离线 Prompt 工具箱，支持 Stable Diffusion WebUI (AUTOMATIC1111)、ComfyUI 和 NovelAI 等主流 AI 绘画工具的用户使用。

项目名称中的 "Rookie" 寓意着帮助新手用户快速上手 AI 绘画，同时也是开发者对自己技术成长的一种期许。

---

## 二、核心功能

### 2.1 魔法解析 🪄 **[已完成 ✅]**

将 AI 生成的图片拖入或点击上传，自动解析图片中嵌入的生成参数。

**支持的生成器：**

- **ComfyUI**：解析 workflow 和 prompt 元数据（通过 KSampler 连接关系精确识别正负提示词）
- **Stable Diffusion WebUI (AUTOMATIC1111)**：解析 parameters 字段
- **NovelAI**：解析 Comment 字段

**解析内容：**

- 正向提示词（Positive Prompt）
- 负向提示词（Negative Prompt）
- 生成参数（采样器、步数、CFG、种子、尺寸等）
- 模型信息（模型名称、哈希值）+ Civitai 跳转链接
- 高清修复参数（放大倍数、重绘幅度、Hires 步数/CFG/Upscaler）
- 调度器（Scheduler）、Clip skip、Version 等扩展参数

**使用流程：**

```
上传图片（支持拖拽/点击/Ctrl+V粘贴） → 自动识别生成器类型 → 提取参数 + 压缩图片 → 展示结果
                                           ↓
                       ┌───────────────────┼───────────────────┐
                       ↓                   ↓                   ↓
                    复制提示词         导入到瀑布画廊        查看历史记录
                                            ↓
                                      （含图片+完整参数）
```

**子组件：**
| 组件 | 功能 | 状态 |
|------|------|------|
| `ImageUploader.vue` | 图片上传（拖拽/点击/粘贴三种方式）+ 全局拖拽遮罩 | ✅ 已完成 |
| `ParseResult.vue` | 解析结果容器（复用 PromptDetailViewer + MetadataViewer） | ✅ 已完成 |
| `HistoryList.vue` | 历史记录列表（时间格式化、选中高亮、单条删除） | ✅ 已完成 |
| `MetadataViewer.vue` | 可折叠 JSON 元数据面板（支持复制原始 JSON） | ✅ 已完成 |

### 2.3 瀑布画廊（提示词管理）📝 **[已完成 ✅]**

集中管理你的提示词库，以**瀑布流卡片**形式展示，支持创建、编辑、删除和快速复制。

**主要功能：**

- ✏️ **创建提示词**：弹窗式创建，支持图片上传 + 名称 + 正负提示词
- 🔄 **编辑修改**：从详情页进入编辑弹窗（使用 PromptDetailForm 可编辑组件）
- 🗑️ **删除管理**：卡片 hover 显示删除按钮 + 弹窗确认
- 🔍 **搜索过滤**：按名称或正向提示词内容模糊搜索
- 📋 **复制提示词**：详情弹窗中可分别复制正向/负向/全部提示词
- 🖼️ **图片关联**：每条提示词可关联预览图（上传时自动压缩为缩略图+预览图）
- 📊 **来源标识**：区分「魔法」（蓝色，来自解析导入）和「手动」（粉色，手动创建）
- 💾 **参数存储**：从魔法解析导入的提示词携带完整的生成参数

**布局特点：**

- 使用 `@yeger/vue-masonry-wall` 实现自适应瀑布流
- 卡片 hover 效果：显示日期/来源标签/名称/删除按钮
- 空状态引导界面

**数据来源：**

- 手动创建（弹窗表单）
- 从魔法解析模块导入（携带图片+完整参数）

**提示词结构：**

```
PromptRecord
├── id (UUID)
├── 名称（如：赛博朋克风格）
├── 正向提示词（期望出现的内容）
├── 负向提示词（期望避免的内容）
├── 关联标签 (TagWithWeight[])
├── 来源标记（'manual' | 'parsed'）
├── 缩略图数据 (Base64, 120x120) -- 用于卡片展示
├── 预览图数据 (Base64, 短边≤900px) -- 用于弹窗大图
├── 生成参数 (ImageParameters)     -- 来自魔法解析导入
├── 创建/更新时间戳
└── ...
```

### 3.3 路由方案

**本项目不使用 Vue Router。** 采用**状态驱动模块切换**方案：

- 在 `App.vue` 中使用 `ref<ModuleType>('spell')` 控制当前活跃模块
- 通过左侧导航栏按钮切换 `currentModule`
- 使用 Vue `<Transition>` 组件做 fade 动画切换
- `ModuleType` 定义为 `'tags' | 'prompts' | 'artists' | 'spell'`

### 3.4 数据存储方案

本项目采用 **IndexedDB (version 3)** 作为本地离线存储方案，所有数据均保存在用户浏览器 IndexedDB 中，不上传至任何服务器。

**数据库配置：**

- 数据库名：`RookieToolsDB`
- 版本：3
- 封装类：`DatabaseService`（单例导出 `db`）

**存储的数据类型：**

| Object Store   | 主键  | 索引                 | 说明                                        | 状态                        |
| -------------- | ----- | -------------------- | ------------------------------------------- | --------------------------- |
| `tags`         | `id`  | category, name       | 七类标签的完整列表                          | ✅ 已启用                   |
| `prompts`      | `id`  | updatedAt            | 用户创建/导入的提示词记录（含 Base64 图片） | ✅ 已启用                   |
| `artists`      | `id`  | isFavorite, useCount | 画师信息和收藏状态                          | ✅ 已启用                   |
| `artistChains` | `id`  | -                    | 画师组合数据                                | ✅ 已启用（模块未接入导航） |
| `parsedImages` | `id`  | createdAt            | 解析过的图片记录（含 Base64 缩略图/预览图） | ✅ 已启用                   |
| `appState`     | `key` | -                    | 应用全局状态                                | ✅ 已启用                   |

**数据安全特性：**

- 完全离线，无需网络连接
- 数据不出本地设备
- 支持浏览器隐私模式
- Proxy 对象深拷贝处理（`JSON.parse(JSON.stringify())` 防 DataCloneError）
- 历史记录限制 50 条（超出自动清理最旧的）
- Blob URL 兼容旧数据的释放机制

## 六、特色设计

### 6.1 离线优先

所有功能无需联网即可使用，数据完全存储在本地浏览器 IndexedDB 中，保护用户隐私。

### 6.2 多生成器兼容

智能识别多种 AI 绘画工具的元数据格式，通过特征字段检测：

1. **workflow/prompt 字段** → ComfyUI（通过 KSampler 连接关系精确定位正负提示词节点）
2. **Software/Comment 包含 novelai** → NovelAI
3. **parameters 字段** → Stable Diffusion WebUI (AUTOMATIC1111)
4. **sd-metadata/DreamStudio** → Stable Diffusion
5. 以上都不匹配 → Unknown（尝试提取通用 ImageDescription）

### 6.3 图片压缩（Canvas API）

解析的图片会自动生成两种尺寸的 Base64 数据：

- **缩略图**：固定 120x120，用于历史列表/卡片展示
- **预览图**：短边自适应（≤900px 时保持原尺寸，否则等比缩放），用于详情弹窗大图展示

有效节省 IndexedDB 存储空间，限制历史记录最多保存 50 条。

### 6.4 使用频率统计

标签和画师都支持使用频率统计（`useCount` 字段）：

- 标签：点击选择时自动递增，排序时按 `useCount` 降序
- 画师：点击卡片名称复制时递增，排序时收藏优先 + `useCount` 降序

### 6.5 模块间集成

魔法解析 → 瀑布画廊 的数据流转：

```
解析完成 → 点击"导入到画廊" → promptStore.importFromMetadata()
                              ↓
              创建 PromptRecord（含 thumbnailData + previewData + parameters）
                              ↓
              写入 IndexedDB prompts store → 自动刷新瀑布流列表
```
