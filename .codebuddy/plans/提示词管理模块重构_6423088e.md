## 产品概述

重构提示词管理模块，将现有的左右布局（左侧列表+右侧编辑器）改为卡片网格布局，支持图片预览和弹窗详情展示。

## 核心功能

1. **布局重构**：移除左侧提示词列表，主内容区改为卡片网格布局
2. **Header操作栏**：新增提示词按钮移至内容区域右上角，支持搜索
3. **卡片展示**：以卡片形式展示提示词预览图（120x120缩略图）、名称、更新时间
4. **弹窗详情**：点击卡片弹出详情弹窗，样式与ParseResult.vue保持一致
5. **图片支持**：

- 从魔法解析导入时自动保留压缩后的图片
- 手动新建/编辑提示词时可上传图片
- 所有图片压缩后存储（复用imageCompressor工具）

6. **公共组件抽离**：将提示词展示区域抽离为可复用组件

## 数据结构变更

PromptRecord新增字段：

- `thumbnailData`: string - Base64缩略图数据(120x120)，用于卡片展示
- `previewData`: string - Base64预览图数据(如果图片短边大于900，则短边缩放到900px，反之不用)，用于弹窗大图展示

## 技术栈

- Vue 3 + TypeScript + Vite
- Pinia状态管理
- Tailwind CSS样式
- IndexedDB本地存储

## 实现方案

### 1. 数据模型扩展

修改 `PromptRecord` 接口，添加 `thumbnailData` 和 `previewData` 字段用于存储压缩后的图片数据。

### 2. 图片压缩复用

复用 `src/utils/imageCompressor.ts` 中的 `compressImage` 函数：

- 卡片展示使用 `thumbnailData` (120x120)
- 弹窗详情使用 `previewData` (原比例压缩)

### 3. 组件抽离策略

新建公共组件 `PromptDetailViewer.vue`：

- 提取 ParseResult.vue 中的提示词展示逻辑
- 支持传入 PromptRecord 数据展示
- 保持与 ParseResult 一致的UI样式（正向/负向提示词卡片、复制按钮等）
- ParseResult 和提示词管理弹窗都复用此组件

### 4. 弹窗交互设计

使用 Vue Teleport + 遮罩层实现弹窗：

- 点击卡片打开详情弹窗
- 弹窗内展示完整提示词信息
- 支持编辑、删除、复制操作

### 5. 数据流设计

```
用户操作(新建/编辑/导入)
    ↓
调用 compressImage 压缩图片（如需要）
    ↓
更新 PromptStore
    ↓
保存到 IndexedDB
    ↓
卡片列表自动更新
```

## 设计风格

保持与现有应用一致的深色/浅色双主题设计，沿用ParseResult.vue的卡片式提示词展示风格。

## 页面结构

### 1. Header区域

- 左侧：模块标题"提示词管理"
- 中间：搜索框（按名称搜索）
- 右侧："+ 新建提示词"按钮（主色调渐变背景）

### 2. 卡片网格区域

- 响应式网格布局：大屏4列、中屏3列、小屏2列
- 卡片内容：
- 顶部：图片预览区（圆角，aspect-ratio保持）
- 中部：提示词名称（单行截断）
- 底部：更新时间 + 来源标签（解析/手动）

### 3. 详情弹窗

- 半透明遮罩层
- 居中弹窗，最大宽度900px，圆角16px
- 复用ParseResult的提示词展示样式：
- 正向提示词：绿色边框卡片
- 负向提示词：红色边框卡片
- 复制按钮带状态反馈
- 底部操作栏：编辑/删除/关闭按钮

### 4. 空状态

- 居中图标 + 文字提示
- "新建提示词"引导按钮

## 交互细节

- 卡片hover：轻微上浮阴影+边框高亮
- 点击卡片：打开详情弹窗
- 弹窗打开/关闭：淡入淡出动画
- 复制按钮：点击后显示"已复制"状态1.5秒
