## 产品概述

将 spell 项目集成为 rookie_tools 的"魔法解析"模块，实现 AI 生成图片的元数据解析功能。

## 核心功能

- **图片拖拽上传**：支持点击上传和拖拽到页面任意位置上传
- **元数据解析**：提取 PNG/JPEG 等格式图片中的 EXIF 和元数据信息
- **生成信息提取**：识别生成器类型（NovelAI、ComfyUI、Stable Diffusion）、分辨率、模型名称、采样器、调度器、CFG Scale、步数、种子
- **提示词解析**：分离并展示正向提示词和负向提示词
- **一键导入**：解析结果可一键导入提示词管理模块
- **解析历史**：保存最近解析的图片记录到 IndexedDB
- **完整元数据**：支持 JSON 格式查看原始元数据

## 视觉要求

采用与现有三个模块一致的现代化卡片式设计，使用 Tailwind CSS 工具类。界面风格保持简洁、专业，与现有三个模块视觉统一。模块采用左右分栏布局：

- **左侧边栏**：历史记录列表，显示最近解析的图片缩略图和文件名
- **右侧主区域**：图片上传区域或解析结果展示

## 技术栈选择

- **前端框架**：Vue 3 + TypeScript + Vite（与现有项目一致）
- **状态管理**：Pinia
- **样式**：Tailwind CSS + Lucide Vue Next 图标
- **图片解析**：ExifReader.js（已验证 spell 项目使用此库）
- **数据存储**：IndexedDB（解析历史记录）

## 实现方案

### 架构设计

采用模块化架构，与现有三个模块保持一致：

- **模块层**：`src/modules/spell-parser/` 包含页面级组件和子组件
- **服务层**：`src/services/spellParser.ts` 封装图片解析逻辑
- **状态层**：`src/stores/spell.ts` 管理解析状态和历史记录

### 核心实现策略

1. **图片解析服务**：封装 ExifReader，支持多格式图片元数据提取，自动识别 ComfyUI workflow、SD WebUI 参数、NovelAI 元数据
2. **解析结果处理**：将解析结果转换为统一的 `ParsedImageData` 格式，包含生成器类型、提示词、参数等字段
3. **历史记录管理**：使用 IndexedDB 存储解析历史，支持查看、删除、重新导入
4. **模块间通信**：解析结果可一键导入提示词管理模块，通过 Pinia Store 实现状态共享

### 数据流设计

```
用户上传图片 → 拖拽/文件选择 → spellParser 服务解析 → Pinia Store 更新状态 → 展示解析结果 → 一键导入提示词管理
```

  id: string;

  positivePrompt: string;
  negativePrompt: string;
  parameters: {
    model?: string;
    sampler?: string;
    scheduler?: string;
    steps?: number;
    cfg?: number;
    seed?: number;
    width?: number;
    height?: number;
  };
  rawMetadata: any;
  createdAt: number;

type ModuleType = 'tags' | 'prompts' | 'artists' | 'spell';

### 性能优化

- 图片预览使用 URL.createObjectURL 避免内存泄漏
- 大图片元数据解析使用异步处理，避免阻塞 UI
- 历史记录分页加载，限制存储数量
- 解析结果缓存，避免重复解析相同图片

采用与现有项目一致的现代化卡片式设计风格，界面简洁专业。模块采用左右分栏布局，与 prompt-manager 模块保持一致的用户体验。

### 页面布局

- **左侧边栏**（w-80）：历史记录列表，显示最近解析的图片缩略图、文件名和解析时间
- **右侧主区域**：图片上传区域或解析结果展示

### 核心区块设计

1. **上传区域**：大型虚线边框拖拽区域，支持点击上传，深色/浅色模式自适应，拖拽时显示视觉反馈
2. **图片预览区**：左侧显示原图预览，右侧显示解析信息，采用响应式布局
3. **信息卡片**：生成器信息、模型信息、采样参数等使用独立卡片展示，使用渐变色标题
4. **提示词区**：正向提示词使用绿色标签，负向提示词使用红色标签，支持一键复制
5. **元数据折叠区**：完整 JSON 元数据可折叠展示，使用代码高亮样式
6. **操作栏**：一键导入提示词管理、复制、保存到历史等操作按钮

### 交互设计

- 拖拽上传时显示视觉反馈（边框高亮、背景变化）
- 解析结果加载使用骨架屏或加载动画
- 复制操作显示成功提示（按钮状态变化）
- 历史记录悬停显示删除按钮
- 使用与现有模块一致的配色方案和圆角风格
