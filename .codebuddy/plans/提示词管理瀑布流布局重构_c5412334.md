## 产品概述

将提示词管理模块的卡片展示从固定网格布局改为瀑布流布局，使图片能够按原比例展示，提升视觉体验。

## 核心功能

1. **瀑布流布局**：使用 vue-virtual-masonry 实现虚拟滚动瀑布流
2. **图片自适应**：移除固定正方形限制，图片按原比例展示
3. **虚拟滚动**：大量数据时保持流畅性能
4. **保持现有功能**：搜索筛选、弹窗详情、编辑删除等交互不受影响

## 技术栈

- Vue 3 + TypeScript + Vite
- vue-virtual-masonry：虚拟滚动瀑布流组件
- Tailwind CSS 样式
- Pinia 状态管理

## 实现方案

### 1. 依赖安装

安装 `vue-virtual-masonry` 包，支持 Vue3 的虚拟滚动瀑布流组件。

### 2. 布局替换策略

- 移除现有的 CSS Grid 布局 (`grid-cols-*`)
- 使用 `<masonry-virtual>` 组件替换网格容器
- 配置列宽、间距、缓冲区等参数

### 3. 卡片样式调整

- 移除 `aspect-square` 固定比例限制
- 图片使用 `h-auto` 按原比例展示
- 保持卡片的圆角、边框、hover 动效

### 4. 虚拟滚动配置

- `columnWidth`: 220px（列宽）
- `gap`: 16px（列间距）
- `buffer`: 5（缓冲区，提前渲染行数）

## 关键实现细节

### Masonry 组件用法

```
<MasonryVirtual
  :items="filteredPrompts"
  :column-width="220"
  :gap="16"
  :buffer="5"
>
  <template #default="{ item }">
    <!-- 卡片内容 -->
  </template>
</MasonryVirtual>
```

### 性能优化

- 虚拟滚动只渲染可见区域，大数据量不卡顿
- 图片懒加载可配合 IntersectionObserver
- 保持组件的 key 稳定，避免不必要的重渲染

## 设计调整

### 布局变化

- 从固定列数的网格改为自适应瀑布流
- 每列宽度固定 220px，自动计算列数
- 卡片高度根据图片比例自适应

### 视觉保持

- 保持深色/浅色双主题
- 卡片样式不变（圆角、边框、阴影）
- Hover 动效保持不变
