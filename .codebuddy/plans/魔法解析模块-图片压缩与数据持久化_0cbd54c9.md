## 产品概述

为"魔法解析模块"实现图片压缩和数据持久化功能，解决当前刷新页面后历史记录图片无法显示的问题。

## 核心功能

1. **图片压缩**：上传图片时自动生成缩略图（120x120）和预览图（800x800），减小存储体积
2. **数据持久化**：将压缩后的图片以 Base64 格式存储到 IndexedDB，刷新页面后仍能正常显示历史记录中的图片
3. **历史记录展示优化**：历史列表使用缩略图，解析结果页使用预览图

## Tech Stack

- 框架：Vue 3 + TypeScript + Pinia
- 存储：IndexedDB
- 图片处理：Canvas API（浏览器原生，无需额外依赖）

## Implementation Approach

### 图片压缩策略

使用 Canvas API 实现客户端图片压缩，生成两种规格：

- **缩略图（thumbnail）**：120x120px，用于历史列表展示
- **预览图（preview）**：800x800px（等比缩放），用于解析结果页展示

压缩流程：

1. 使用 `FileReader` 读取图片文件
2. 创建 `Image` 对象加载图片
3. 计算等比缩放后的尺寸
4. 使用 Canvas 绘制并导出为 JPEG（质量 0.85）
5. 转换为 Base64 字符串存储

### 数据持久化策略

修改 `ParsedImageData` 数据结构：

- 新增 `thumbnailData: string` - Base64 缩略图数据
- 新增 `previewData: string` - Base64 预览图数据
- 移除/废弃 `previewUrl: string` - 改用动态生成的内存 URL

展示时：

- 从 IndexedDB 读取 Base64 数据
- 使用 `data:image/jpeg;base64,xxx` 格式直接展示
- 或临时创建 Blob URL（刷新后从 Base64 重新生成）

### 性能优化

- 压缩质量设置为 0.8，平衡画质与体积
- 最大预览尺寸限制为 800px，避免超大图片占用过多内存
- IndexedDB 存储 Base64 字符串，展示时直接使用，无需额外请求

### 数据迁移

- 由于数据结构变更，旧数据中的 `previewUrl` 会失效
- 历史记录列表对无效图片显示占位图标即可
- 新上传的图片使用新数据结构
